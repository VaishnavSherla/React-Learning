Setup

  • Initialize Project: npm init - Create a new Node.js project and generate a package.json file.
  • Install Parcel (Development Dependency): npm i -D parcel - Install Parcel as a development dependency, a web application bundler.
  • Install React and ReactDOM: npm i react react-dom - Install React and ReactDOM for building user interfaces.
  • Exec using npx > Start Development Server: npx parcel index.html - Use Parcel to bundle and serve your app with index.html as the entry point.
  • Build for Production: npx parcel build index.html - Create a production build without specifying an entry point.
  • Start Development Server with HTTPS: npx parcel index.html --https - Run the dev server with HTTPS enabled.

Dependencies

Dependencies are the packages your application relies on:

  • React: npm i react - Library for building user interfaces.
  • ReactDOM: npm i react-dom - For rendering React components to the DOM.

DevDependencies

DevDependencies are packages used for development and testing:

  • Parcel (as a devDependency): npm i -D parcel - A web application bundler used for development.

Transitive Dependencies

Transitive dependencies are packages that are not explicitly listed in your package.json but are required by your direct dependencies:

  • These are managed automatically by npm to ensure your application's dependencies are met.
  • No direct action is needed to manage transitive dependencies.

Browserslist Configuration From Babel

The browserslist configuration specifies the browsers your project should support for compatibility. This configuration can be used by various tools, including Babel, to determine transpilation and CSS prefixing. It helps ensure that your web application functions correctly and looks consistent across different browsers.

What is Browserslist?

Browserslist is a popular configuration file used in web development to specify a list of target browsers and their versions. It provides a clear way to define the browsers you want your project to support and helps developers and build tools make decisions about transpiling and optimizing code for specific browser versions.

Example Configuration

Here's an example of a browserslist configuration that specifies targeting the last two versions of the Chrome browser:

    "browserslist": ["last 2 Chrome versions"]

In this example:

  • "last 2 Chrome versions" indicates that your project should support the most recent two versions of the Chrome browser.
  • You can customize this configuration to target different browsers and versions based on your project's requirements.

Usage with Babel

When you use Babel for JavaScript transpilation, it can use the browserslist configuration to determine which ECMAScript (JavaScript) features to transpile or polyfill based on your target browsers. This ensures that your code remains compatible with the specified browsers.

Other Tools

In addition to Babel, other tools like Autoprefixer (for CSS prefixing) and PostCSS also make use of the browserslist configuration to apply appropriate transformations to your code.

By defining your target browsers in the browserslist configuration, you can ensure a smoother cross-browser development experience and improved compatibility for your web projects.

Tree Shaking

Tree Shaking is a process in modern JavaScript bundlers (including Parcel) that eliminates unused code from your final bundle. It helps reduce the size of your JavaScript bundle by removing any code that is not actually used in your application. This is crucial for optimizing your application's performance and decreasing load times, especially for large web applications.

About Babel

Babel is a popular JavaScript compiler or transpiler that allows you to write modern JavaScript code (including ES6 and beyond) and transpile it into JavaScript code that can run in older browsers or environments. Babel is often used in modern web development to ensure cross-browser compatibility and to take advantage of the latest language features without worrying about browser support.

When you configure Babel with your project, it can use the browserslist configuration to determine which JavaScript features to transpile or polyfill based on your target browsers. This way, Babel ensures that your code remains compatible with the specified browsers, allowing you to write modern JavaScript while still supporting a wide range of user environments.

By using Babel and the browserslist configuration, you can confidently write modern JavaScript code and have it automatically transformed for compatibility with your target browsers.
