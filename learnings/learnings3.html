React One-Way Data Flow

React is beast at re rendering the components

React, a popular JavaScript library for building user interfaces, follows a one-way data flow model:

  • Data Flow Direction: Data flows from parent components to child components.
  • we have child to parent also this is known as liftig state up.
  • Parent-Child Relationship: In React, components are organized hierarchically with parent components containing child components.
  • Data Passing: Data is passed from parent components to child components as props.
  • Rendering: Child components use the received data as props to render their part of the UI.
  • Callback Functions: Child components can send data back to parent components by invoking callback functions passed as props.
  • Local State: React components can also manage local state using hooks(normal js func) (e.g., useState) or class component state.
  • Reactivity: Changes to state trigger component re-renders, ensuring that the UI stays in sync with the data.
  • Top-Down Approach: Data typically flows from higher-level (parent) components to lower-level (child) components in a top-down manner, making data flow predictable and easier to trace.
as soon as the local state var updates React will re render the component so that makes the websites faster ( dom manipulation is fast)

While React primarily follows one-way data flow, it doesn't inherently support two-way data binding like Angular. Developers can achieve similar behavior to two-way data binding for form inputs by manually controlling input values and state updates.


<input
  type="text"
  value={inputValue}
  onChange={handleInputChange}
/>

In the above example, the input's value is controlled (read from state) and updated (via the onChange event handler) manually, giving developers control over the data flow.

React Working Behind the Scenes

React uses a reconciliation algorithm known as React Fiber.

ability to split rendering work into chunks and spread it out over multiple frames

How does it work?

React operates by creating a virtual DOM (not the actual DOM, but a representation of it, in the form of React elements, which are JavaScript objects).

For example:

console.log(<Body/>)

React then employs a diffing algorithm to identify differences between the updated or new virtual DOM and the old virtual DOM.

Finally, it updates the actual DOM accordingly.
